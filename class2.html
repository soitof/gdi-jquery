Events!

Events (actions) that occur within the DOM.

These events let event-driven languages register various event handlers/listeners.

Events get sent down from the window to the element, which then bubbles it back up.

Handlers will catch the event and deal with it.

There is a huge collection of events the dom can handle.

  • Mouse events
  • Keyboard events
  • HTML frame/object events
  • HTML form events
  • User interface events
  • Mutation (structural) changes

Common events

Keyboard events: keyup, keydown, keypress

Mouse events: click, mousedown, mouseup

Form events: change, focus, blur

Add some events


$('#button').on('click', function () {
	console.log('clicked!');
});
					

$('form').on('submit', function (event) {
    event.preventDefault();
    console.log('Not submitting, time to validate!');
});

Callbacks

A piece of executable code that is passed as an argument to other code.

Code accepting the call back executes it at a convenient time.

This can be immediate, a synchronous callback, or it might happen later, an asynchronous callback.

Events + Callbacks

Callbacks are perfect for event handlers.

Give the event handler code to execute when the event is captured.

This can either be done inline, or the function can be defined before hand.

Adding callbacks


$('form').on('submit', function (event) {
	event.preventDefault();
    console.log('Not submitting, time to validate!');
});
					

function onButtonClick() {
	console.log('clicked!');
};

$('#button').on('click', onButtonClick);

Effects and animations

jQuery provides support and techniques for adding animations and effects to your page.

This includes simple, standard animations.


$('#error').toggle(1000);
					

$('#error').fadeIn();
					

$('#error').show(1000, function(){
        $(this).addClass('redText')
});

Get fancy with your own.

You also have the ability to create your own.


$(selector).animate({params},speed,callback);

This moves a

element to the right, until it has reached a left property of 250px.


$(selector).animate({params},speed,callback);

The required params parameter defines the CSS properties to be animated.

Speed specifies the duration of the effect: "slow", "fast", or milliseconds.

The callback parameter is a function to be executed after the animation

Plugins

Similar to libraries.

Adds a specific feature to an existing code.

  • Code someone else has written.
  • Included, or injected, with your code.
  • Documented for use.

"If you want to create an animation, effect, or UI component, chances are pretty good that someone has done the work for you already."

Which should I use?

http://blog.pamelafox.org/2013/07/which-javascript-library-should-i-pick.html

How would I use one?

Download the plugin and associated files from the site or github repo.

Copy them into your project folder.

You may need to reference certain items on to your page.

Perhaps a css file.


<link rel="type/stylesheet" type="text/css" href="tablesorter.css">

In the HTML, add a <script> tag for the jQuery plugin itself.


<script src="lib/tablesorter.js"><script>

In the JavaScript, call the jQuery plugin on your DOM.


$('table').tableSorter();

Patterns

No one can tell you how to code.

But we don't want everyone to hate us.

There are "rules" to follow known as patterns.

Remember no one can tell you to stick to these rules.

Your friends, coworkers, other people who read your code, will appreciate it.

Some jQuery patterns

Name variables with $


var $myVar = $('#myNode');

Bind events to the document.


$(document).on('click', 'a', myCallback);

Store references to callback functions.


var myCallback = function(argument) {
  // do something cool
};

$(document).on('click', 'p', myCallback);

Descend from the id selector.


var arms = $('#container div.robotarm');

Ying and Yang

Just like good practices, there are bad ones too.

There are styles to avoid.

Once again, there is nothing wrong with these styles.

It is simply that there are better ways. Your code might run faster, be safer, etc...

The choice is still yours.

Some jQuery anti-patterns

Requerying all the time.


$('body').append('<div class="stuff"/>');
$('div.stuff').click(function(){});

Using a universal selector.


$('.class :radio') // implied universal
					

$('.class *:radio') // explicit universal
					

$('.class input:radio') // much better

A new var for ever variable.


var one = 1;
var two = function() {//stuff};
var three = two(one);
					

var one = 1,
	two = function() {//stuff},
	three = two(one);

Keep the chain going.

As part of not requerying you can chain methods on the end of one another.


banner.css('color', 'red');
banner.html('Welcome!');
banner.show();

This could also be written as


banner.css('color', 'red').html('Welcome!').show();

If you're real fancy.


banner.css('color', 'red');
	  .html('Welcome!');
	  .show();

DOM readiness

Broken code

DOM manipulation and event binding in the <head>


<head>
 <script>
  $('body').append( myNode );
 </script>
</head>

If you need to load a script in the <head>, wait for the rest of the DOM to be ready before executing.

You can do this with events.

The ready event can handle this.


<body>
 ...
 <script>
  $(document).ready(function() {
    // do something when the DOM is fully loaded
  });
 </script>
</body>

Do you really need it?

jQuery is very nice, but do you need it?

Remember you can do all the things you do with jQuery with pure JavaScript.

jQuery as part of all that it adds, may give you a lot more than you need.

You are always loading the entire library when you use it.

This can be slow and cumbersome.

youmightnotneedjquery.com/

Look at all of your options.

Make sure it suits your needs best.
